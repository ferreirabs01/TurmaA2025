Adeus, LiveServer!

Transformando o Node.js em um Servidor de Arquivos Completo

"Por que usar uma extensão se o seu próprio Backend pode servir o Front-end?"

O que são Arquivos Estáticos?

São arquivos que o servidor envia para o navegador **exatamente como estão no disco**, sem processamento dinâmico.

  • HTML: A estrutura do seu site.
  • CSS/JS: O estilo e comportamento.
  • Imagens: Seu logo, ícones e banners.

A Analogia da Janela de Atendimento

Imagine o seu servidor Node.js como uma **Fábrica Fechada** (Segurança).

Ninguém pode entrar lá dentro e pegar suas receitas (Código Fonte).

Ao expor uma pasta public, você está criando uma **Janela de Atendimento**.

O cliente só pode tocar e pegar o que você coloca naquela janela específica.

LiveServer vs Produção

O PROBLEMA:

O LiveServer cria um servidor separado (ex: porta 5500). Se seu Backend está na 3000, você terá erros de CORS e problemas ao subir para um Windows Server real.

A SOLUÇÃO:

O Node.js assume o controle. O Backend e o Front-end rodam na **mesma porta**, como uma aplicação profissional única.

Passo 1: Estrutura de Pastas

Crie a pasta public na raiz do seu projeto:


MeuProjeto/
├── app.js
├── package.json
└── public/          <-- Janela de Atendimento
    ├── index.html
    ├── style.css
    └── img/
        └── logo.png

Passo 2: Expondo a Pasta no Node

Use o middleware nativo do Express express.static:


const express = require('express');
const path = require('path');
const app = express();

// A MÁGICA ACONTECE AQUI:
// Dizemos ao Express que tudo na pasta 'public' é livre para acesso
app.use(express.static('public'));

// Opcional: Servir a pasta de imagens com um prefixo
app.use('/imagens', express.static('img'));

app.listen(3000, () => {
    console.log('Servidor rodando na porta 3000');
    console.log('Acesse seu site em: http://localhost:3000');
});

Missão: Independência

  1. Feche o LiveServer do seu VS Code.
  2. Crie a pasta public e mova seu HTML/CSS para lá.
  3. Adicione o app.use(express.static('public')) no seu Backend.
  4. Tente acessar o site via porta do Node (ex: 3000).

Servidor Unificado!

Agora seu Backend é um servidor web completo e profissional.
