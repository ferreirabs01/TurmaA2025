Middlewares

O "Porteiro" e o "Filtro" das suas Requisições

"Middleware é tudo o que acontece entre o pedido do cliente e a resposta final do servidor."

O que é um Middleware?

No Express, um middleware é uma função que tem acesso aos objetos de:

  • Request (req): O que o cliente enviou.
  • Response (res): O que o servidor vai responder.
  • Next: A função que passa a bola para o próximo passo.

Analogia: O Segurança da Balada

Imagine uma Balada (sua Rota de API):

  1. Middleware 1 (Segurança): Verifica se você está com RG e se tem idade permitida. Se não tiver, ele te barra ali mesmo (Response 403).
  2. Middleware 2 (Revista): Verifica se você está portando algo proibido. Se estiver OK, ele diz: "Next!".
  3. Destino Final (Pista): Você finalmente entra e curte a festa (Sua lógica de código).

Onde eles atuam?

1. Nível de Aplicação: Afeta todas as rotas (ex: `app.use(express.json())`).

2. Nível de Rota: Atua apenas em um caminho específico (ex: verificar se o usuário é Admin antes de deletar algo).

3. Tratamento de Erros: Captura falhas e envia uma resposta amigável.

Exemplo 1: Logger (Global)

Este middleware registra cada acesso no terminal:


app.use((req, res, next) => {
    console.log(`[${new Date().toISOString()}] ${req.method} em ${req.url}`);
    next(); // Fundamental para não travar a requisição!
});

Exemplo 2: Autenticação (Específico)


const verificarSenha = (req, res, next) => {
    const { senha } = req.headers;
    if (senha === 'admin123') {
        next(); // Senha correta, pode passar
    } else {
        res.status(403).send('Acesso Negado!');
    }
};

// Aplicando apenas na rota de exclusão:
app.delete('/usuario/:id', verificarSenha, (req, res) => {
    res.send('Usuário deletado!');
});

Missão: O Filtro Neon

  1. Crie um middleware que verifique se o corpo da requisição (body) está vazio.
  2. Se estiver vazio, responda com erro 400.
  3. Se tiver dados, deixe passar usando next().
