Swagger & OpenAPI

A Vitrine da sua API Backend

"Se o código é o motor, o Swagger é o painel de controle do piloto."

A Analogia do Cardápio

Imagine um restaurante sem cardápio:

  • O cliente não sabe o que pedir.
  • O garçom fica exausto de explicar.
  • Erros de pedido acontecem toda hora.

Com Swagger:

  • O Front-end vê todos os pratos (Rotas).
  • Sabe os ingredientes (Parâmetros).
  • Sabe o preço (Tipo de Retorno).

OpenAPI vs Swagger

OpenAPI: É a Regra/Padrão. Um documento (JSON ou YAML) que descreve como a API funciona.

Swagger: É o Conjunto de Ferramentas que pega esse padrão e cria a interface visual bonita e interativa que usamos.

Passo 1: Preparando o Terreno

No seu projeto Node.js, instale as bibliotecas necessárias:


                npm install swagger-ui-express swagger-jsdoc
swagger-ui-express: Cria a página visual.
swagger-jsdoc: Lê seus comentários no código e gera a documentação.

Passo 2: Configurando o Motor

Adicione isso ao seu app.js ou server.js:


const swaggerJsDoc = require('swagger-jsdoc');
const swaggerUi = require('swagger-ui-express');

const swaggerOptions = {
    swaggerDefinition: {
        openapi: '3.0.0',
        info: {
            title: 'API de Alunos - Lab Backend',
            version: '1.0.0',
            description: 'Sistema de Gestão de Alunos 2026'
        },
        servers: [{ url: 'http://localhost:3000' }]
    },
    apis: ['app.js'] // Onde o Swagger vai procurar os comentários
};

const swaggerDocs = swaggerJsDoc(swaggerOptions);
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(swaggerDocs));

Passo 3: Documentando uma Rota

Use o padrão JSDoc acima da sua rota:


/**
 * @openapi
 * /alunos:
 * get:
 * description: Retorna a lista completa de alunos.
 * responses:
 * 200:
 * description: Sucesso ao retornar a lista.
 */
app.get('/alunos', (req, res) => {
    res.status(200).send({ id: 1, nome: 'Estudante Neon' });
});

A Magia do "Try it out"

Diferente de um PDF estático, o Swagger permite que o desenvolvedor:

  • Execute rotas diretamente do navegador.
  • Veja o JSON de resposta em tempo real.
  • Teste códigos de erro (404, 500) sem precisar do Postman.

Observações de Segurança

⚠️ Atenção em Produção!

Em ambiente real, nunca deixe sua documentação aberta para qualquer um. Use um Middleware de autenticação para proteger a rota /api-docs.
