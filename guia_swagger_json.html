Swagger via JSON

Organização Máxima: Separando a Documentação do Código

"Mantenha seu código limpo e sua documentação centralizada em um único lugar."

Por que usar JSON externo?

  • Código Limpo: Suas rotas Node.js ficam livres de comentários gigantes.
  • Portabilidade: O arquivo JSON pode ser importado em outras ferramentas (Postman, Insomnia).
  • Padronização: Facilita o trabalho em equipe onde alguém foca só na escrita da especificação.

Passo 1: Crie o arquivo swagger.json

Na raiz do projeto, defina toda a estrutura da API:


{
  "openapi": "3.0.0",
  "info": {
    "title": "Minha API Backend",
    "version": "1.0.0"
  },
  "paths": {
    "/usuarios": {
      "get": {
        "summary": "Lista todos os usuários",
        "responses": {
          "200": { "description": "OK" }
        }
      }
    }
  }
}

Passo 2: Carregando no Node.js

Basta importar o JSON e passar para o middleware:


const express = require('express');
const swaggerUi = require('swagger-ui-express');
const swaggerDocument = require('./swagger.json'); // Importa o arquivo

const app = express();

// Rota da documentação
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(swaggerDocument));

app.listen(3000, () => console.log('Online!'));

A Analogia da Planta do Prédio

Enquanto o JSDoc é como colocar etiquetas em cada tomada enquanto você constrói a casa...

...o JSON Externo é como ter a **Planta Azul (Blueprint)** em uma mesa separada. Você consulta o mapa completo sem precisar abrir as paredes.

Dica Pro: Inteligência no JSON

No VS Code, se você começar o arquivo com a chave "$schema", ele te dará auto-complete para não errar as tags da OpenAPI.


{
  "$schema": "https://json.schemastore.org/swagger-2.0.json",
  ...
}

Missão JSON

  1. Crie um arquivo swagger.json.
  2. Remova os comentários JSDoc das suas rotas.
  3. Refatore o app.js para ler o arquivo externo.
  4. Tente adicionar um novo endpoint de "Login" diretamente no JSON.
