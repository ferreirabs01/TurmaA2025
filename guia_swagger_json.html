Swagger via JSON

Organização Máxima: Documentação Separada do Código

"Mantenha seu código limpo e sua especificação centralizada em um único arquivo."

A Planta do Prédio

Se documentar via JSDoc é como colocar etiquetas em cada tomada enquanto constrói a casa...

...o JSON Externo é como ter a **Planta Azul (Blueprint)** em uma mesa separada. Você consulta o mapa completo sem precisar abrir as paredes.

Passo 0: Instalação

Mesmo usando um arquivo externo, precisamos da biblioteca que renderiza a interface no Express:


                npm install swagger-ui-express
Nota: Neste modo (JSON externo), não precisamos da biblioteca swagger-jsdoc, pois escreveremos o JSON manualmente.

Passo 1: Criar o JSON

Crie um arquivo chamado swagger.json na raiz do seu projeto:


{
  "openapi": "3.0.0",
  "info": {
    "title": "Minha API Backend",
    "version": "1.0.0",
    "description": "Documentação via arquivo JSON externo"
  },
  "paths": {
    "/usuarios": {
      "get": {
        "summary": "Lista todos os usuários",
        "responses": {
          "200": { "description": "Lista enviada com sucesso" }
        }
      }
    }
  }
}

Passo 2: Configurar a Rota

No seu app.js, importe o arquivo e configure o middleware:


const express = require('express');
const swaggerUi = require('swagger-ui-express');
const swaggerDocument = require('./swagger.json'); // Importando o arquivo

const app = express();

// Configuração da rota visual
app.use('/api-docs', swaggerUi.serve, swaggerUi.setup(swaggerDocument));

app.get('/usuarios', (req, res) => {
    res.json([{ id: 1, nome: 'Dev Neon' }]);
});

app.listen(3000, () => console.log('Acesse: http://localhost:3000/api-docs'));

Por que este método?

  • Código Limpo: Suas rotas ficam livres de comentários gigantes.
  • Portabilidade: O arquivo JSON pode ser usado no Postman ou Insomnia.
  • Colaboração: Outros membros da equipe podem editar a documentação sem tocar no código lógico.

Missão JSON

  1. Crie o arquivo swagger.json.
  2. Remova os comentários JSDoc das suas rotas (se houver).
  3. Faça o require do JSON no seu servidor.
  4. Adicione uma nova rota de **POST /login** diretamente no arquivo JSON e salve para ver a atualização.
